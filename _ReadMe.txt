1. 该项目为自毁与自备份
2. 主要的执行脚本位于run中, 其中的脚本会尝试自毁
3. run中的第一个执行的脚本由该文件夹的脚本启动. 
  启动前会对run中的文件内容进行备份, 存储到backup
  文件夹中. 


已解决的问题:

1. 解决了文件的复制问题
   1) copy
   2) xcopy
2. 解决了删除问题
   1) del
   2) rmdir
3. 解决了切换路径问题
   1) cd \d %~dp0
4. 解决了乱码问题
   1) 乱码由注释产生
   2) 如果编码为ANSI则问题解决. 
5. 解决了空run 文件夹时backup 也被清空的问题.
   1) 不使用清空backup 的命令, 而是以覆盖的形式复制文件
   2) 设置标识符%flag%参数, 用if 语句判断backup 是否为空
   3) 如果判断backup 为空, 则显示错误提示信息, 推出执行
6. 解决判断文件夹是否为空问题
   1) 遍历文件夹目录下的文件, 设置标识符flag
   2) 如果文件夹为空, 则循环次数为0, flag=0
   3) 如果文件夹不为空, set flag=1. 
7. ||, &&, % 的用法
8. 参数的设置
   1) set varname=value
   2) 上述命令等号左右侧不可加空格
   3) 用%varname% 来引用设置好的参数.
